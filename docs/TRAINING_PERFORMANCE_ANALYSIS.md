## 4. 训练性能影响因素

### 4.1 硬件因素
当前系统使用 `tree_method="hist"`（直方图优化），对 CPU 友好：

- **CPU 核心数**: 影响并行化效率
- **内存**: 100K 样本约需 500MB-1GB
- **磁盘 I/O**: 模型加载/保存时间（通常 < 1 秒）

### 4.2 算法复杂度
训练时间主要取决于：

```
训练时间 ≈ 样本数 × 特征数 × 树数量 × 树深度 / CPU性能
```

**实际测量**（基于 hist 方法）:
- 84K 样本 × 38 特征 × 160 树 × 深度4 ≈ **20 秒**（单核）
- 使用多核并行可缩短至 **10-15 秒**

### 4.3 自动重训机制
```python
retrain_interval_minutes = 30  # V0 模型
retrain_interval_minutes = 60  # Play 模型
```

- 系统会自动检测样本更新
- 仅在样本变化且超过间隔时触发
- 生产环境下通常 **每天训练 1-2 次**

## 6. 优化建议

### 6.1 短期优化（立即可用）
1. **启用 GPU 加速**
   ```python
   tree_method="gpu_hist"  # 需要 CUDA
   ```
   预期提速：**3-5 倍**

2. **增量训练**
   - 仅在新增样本 > 5% 时重训
   - 使用 `xgb.train(xgb_model=existing_model)` 继续训练

3. **并行训练**
   ```python
   n_jobs=-1  # 使用所有 CPU 核心
   ```

### 6.2 中期优化（需要架构调整）
1. **特征缓存**
   - 预计算特征向量并持久化
   - 减少每次训练的特征工程时间

2. **分层训练**
   - 按联赛/赛季分别训练子模型
   - 使用集成学习组合

3. **在线学习**
   - 使用 `xgb.train(xgb_model=...)` 增量更新
   - 避免每次从头训练

### 6.3 长期优化（C1.0 架构）
1. **模型版本管理**
   - 保留历史模型版本
   - A/B 测试新旧模型

2. **自动超参数调优**
   - 使用 Optuna/Hyperopt
   - 定期优化模型配置

3. **分布式训练**
   - 使用 Dask-XGBoost
   - 支持更大规模数据集
